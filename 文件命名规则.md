**1. 根项目名**



建议：



global-site-factory



不要叫：



website

cms

frontend

company-sites



因为你的项目本质是“多实体公司官网生成平台”。



**2. 包名命名**



统一用：



@factory/\*



例如：



@factory/site-builder

@factory/ui

@factory/theme-enterprise

@factory/seo

@factory/validators

@factory/site-core

**3. 公司站点命名**



公司站点目录用小写短横线：



sites/acme-manufacturing

sites/beta-saas

sites/gamma-energy



不要用：



AcmeManufacturing

acme\_manufacturing

acme manufacturing

company1

siteA

**4. 文件命名规则**



建议这样：



组件文件：PascalCase.tsx

例如：Hero.tsx、ServicesGrid.tsx



工具文件：kebab-case.ts

例如：load-site.ts、build-sitemap.ts



schema 文件：\*.schema.ts

例如：site-config.schema.ts、page.schema.ts



脚本文件：kebab-case.ts（仓库根脚本在 `scripts/`，站点脚手架脚本在 `tools/create-site/`，对外 CLI 在 `tools/site-cli/`）

例如：`scripts/site-pipeline.ts`、`tools/site-cli/site.ts`、`tools/create-site/create-site.ts`、`scripts/build-site.ts`。

说明：脚手架**不要**误以为存在 `scripts/create-site.ts`。若仓库中有该文件名，其为误路径提示桩；真实脚手架在 `tools/create-site/` 且必须通过 **`pnpm site create <siteSlug>`** 调用。



内容文件：kebab-case.json

例如：home.json、about.json、contact.json
